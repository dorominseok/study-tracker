# Study Tracker 보고서 

Study Tracker는 PC 환경에서 학습 시간을 기록하고 시각화하여 사용자가 자신의 공부 패턴을 파악하고 꾸준히 학습할 수 있도록 돕는 웹 기반 학습 관리 시스템이다.

## 1. 실험의 목적과 범위

### 1.1 실험의 목적

본 프로젝트는 사용자가 자신의 공부 시간을 기록하고 관리하며 학습 현황을 한눈에 확인할 수 있도록 웹 기반 학습 관리 시스템을 구현하는 것을 목표로 한다. 기존의 단순한 타이머 기능에서 나아가 공부 시작, 일시정지, 재개, 종료까지의 과정을 하나의 학습 세션으로 관리하고, 이를 데이터베이스에 저장하여 학습 기록을 체계적으로 활용할 수 있도록 구성하고자 하였다.

또한 기록된 학습 데이터를 바탕으로 일일 공부 시간, 최근 7일 학습 시간, 월별 히트맵 통계와 같은 시각화 기능을 제공하여 사용자가 자신의 공부 패턴을 보다 직관적으로 파악할 수 있도록 하였다. 그리고 오늘의 목표 공부 시간을 설정하고 달성 여부를 확인할 수 있는 기능을 통해 자기주도적인 학습 관리를 지원하는 것을 목적으로 한다.

### 1.2 실험의 범위

본 프로젝트에서는 다음과 같은 기능을 구현 범위에 포함하였다.

- 회원가입 및 로그인 기능
- JWT 기반 인증 처리
- 과목 생성, 조회, 삭제 기능
- 공부 타이머 시작, 일시정지, 재개, 종료 기능
- 공부 기록 저장 및 조회 기능
- 일일 통계, 최근 7일 통계, 월별 히트맵 통계 기능
- 오늘 목표 공부 시간 설정 및 최근 목표 달성 기록 조회 기능
- 탭 이탈 감지 및 자동 일시정지 기능
- 타이머 일시 정지시 알람 기능

다음 항목은 본 프로젝트의 구현 범위에서 제외하였다.

- 강제 화면 Lock 기능
- Fullscreen 기반 집중 모드
- 시간별 통계 및 학습패턴 시각화
- 과목별 상세 통계와 기간별 고급 필터링
- 배포 환경 구성

아래 그림은 Study Tracker 시스템에서 비회원과 회원이 수행할 수 있는 주요 기능을 나타낸 유스케이스 다이어그램이다.

![유스케이스 다이어그램](docs/diagrams/usecase.png)

비회원은 회원가입과 로그인을 수행할 수 있으며, 회원은 로그인 이후 과목 관리, 공부 시작 및 종료, 기록 조회, 통계 조회, 목표 관리 등의 기능을 사용할 수 있다.

### 2.3 기능 명세

#### 1. 회원가입 및 로그인

사용자는 이메일과 비밀번호를 입력하여 회원가입을 진행할 수 있으며, 로그인 후에는 JWT 토큰을 발급받아 인증이 필요한 기능을 사용할 수 있다.

주요 내용
- 이메일과 비밀번호를 이용한 회원가입 및 로그인
- 로그인 성공 시 JWT 토큰 발급
- 토큰 기반 인증 처리

#### 2. 과목 관리

사용자는 자신의 계정에 과목을 등록하고, 등록된 과목 목록을 조회하거나 필요하지 않은 과목을 삭제할 수 있다. 생성된 과목은 공부 세션과 연결되어 기록을 보다 체계적으로 관리할 수 있도록 한다.

주요 내용
- 과목 생성
- 과목 목록 조회
- 과목 삭제
- 공부 세션과 과목 연결

#### 3. 공부 타이머

공부 타이머는 공부 시작, 일시정지, 재개, 종료의 흐름을 하나의 세션으로 관리하는 기능이다. 사용자는 과목을 선택하고 메모를 남기며 공부를 기록할 수 있고, 타이머 종료 시 기록은 자동으로 저장된다.

주요 내용
- 공부 시작, 일시정지, 재개, 종료
- 과목 선택 및 메모 기록
- 세션 상태 관리
- 종료 시 공부 기록 자동 저장

#### 4. 통계 및 시각화

저장된 공부 기록을 바탕으로 오늘 공부 시간, 최근 7일 학습 시간, 월별 히트맵 통계를 제공한다. 이를 통해 사용자는 자신의 학습량과 공부 패턴을 직관적으로 확인할 수 있다.

주요 내용
- 일일 공부 시간 조회
- 최근 7일 학습 시간 조회
- 월별 히트맵 통계 제공
- 메인화면과 통계 화면에서 시각화 결과 확인

#### 5. 목표 설정

사용자는 하루 목표 공부 시간을 설정하거나 수정할 수 있으며, 시스템은 실제 공부 시간과 비교하여 목표 달성 여부를 함께 보여준다. 최근 목표 기록도 함께 제공하여 계획과 결과를 비교할 수 있도록 구성하였다.

주요 내용
- 오늘 목표 공부 시간 저장 및 수정
- 목표 달성 여부 계산
- 최근 목표 기록 조회

#### 6. 집중 보조 기능

공부 중 브라우저 탭을 일정 시간 이상 벗어나면 자동 일시정지 기능이 동작하여 실제 학습 시간만 반영되도록 한다. 자동 일시정지 발생 시에는 알림 메시지를 통해 현재 상태를 사용자에게 안내한다.

주요 내용
- 탭 이탈 감지
- 자동 일시정지 처리
- 자동 일시정지 알림 표시

## 3. 설계

### 3.1 시스템 구조 설계

본 프로젝트는 클라이언트-서버 구조로 설계하였으며, 사용자는 React 기반 프론트엔드에서 화면을 통해 기능을 이용한다. 프론트엔드는 Express 서버의 REST API를 호출해 데이터를 주고받고, 서버는 인증 처리와 비즈니스 로직, 데이터베이스 연동을 맡는다. 모든 기록 데이터는 MariaDB에 저장된다.

아래 그림은 본 프로젝트의 전체 시스템 구조를 나타낸다.

![시스템 구조도](docs/diagrams/system-architecture.png)

시스템 구조는 크게 다음 세 부분으로 구분된다.

- 클라이언트 계층: React 기반 사용자 인터페이스
- 서버 계층: Express 기반 API 서버
- 데이터 계층: MariaDB 기반 학습 기록 저장소

### 3.2 클래스 다이어그램

본 프로젝트는 객체지향 클래스 중심 구조보다는 데이터 중심의 웹 서비스 구조이므로, 클래스 다이어그램은 핵심 도메인 엔티티와 데이터 관계를 중심으로 설계하였다. 핵심 클래스는 `User`, `Subject`, `StudySession`, `DailyGoal` 4개로 구성된다.

![클래스 다이어그램](docs/diagrams/class-diagram.png)

각 클래스의 역할은 다음과 같다.

- `User`
  - 사용자 계정 정보를 저장한다.
  - 주요 속성: `id`, `email`, `password_hash`
- `Subject`
  - 사용자별 과목 정보를 저장한다.
  - 주요 속성: `id`, `user_id`, `name`, `created_at`
- `StudySession`
  - 공부 시작 시각, 종료 시각, 누적 시간, 메모, 상태 정보를 저장한다.
  - 주요 속성: `id`, `user_id`, `subject_id`, `start_time`, `end_time`, `duration`, `memo`, `status`, `last_resume_time`, `created_at`
- `DailyGoal`
  - 사용자별 하루 목표 공부 시간을 저장한다.
  - 주요 속성: `id`, `user_id`, `target_minutes`, `created_at`

클래스 간 관계는 다음과 같다.

- `User` 1 : N `Subject`
- `User` 1 : N `StudySession`
- `User` 1 : N `DailyGoal`
- `Subject` 1 : N `StudySession`

단, `StudySession`의 `subject_id`는 선택값이므로 과목을 지정하지 않은 공부 세션도 생성할 수 있다.

### 3.3 순서 다이어그램

순서 다이어그램은 사용자의 주요 기능 수행 과정에서 프론트엔드, 백엔드, 데이터베이스가 어떤 순서로 상호작용하는지를 설명하기 위해 작성하였다. 각 순서 다이어그램은 사용자 요청이 프론트엔드, 백엔드, 데이터베이스를 거쳐 처리되는 과정을 시간 순서에 따라 나타낸다.

#### 3.3.1 로그인 순서 다이어그램

사용자가 로그인 정보를 입력하면 프론트엔드가 로그인 API를 호출하고, 서버는 DB에서 사용자 정보를 조회한 후 비밀번호를 검증한다. 검증이 완료되면 JWT 토큰을 생성하여 프론트엔드에 반환하고, 프론트엔드는 이를 저장하여 이후 인증 요청에 사용한다.

로그인 성공 후 프론트엔드는 발급받은 JWT 토큰을 저장하고, 이후 인증이 필요한 API 요청의 `Authorization` 헤더에 포함하여 전송한다.

![로그인 순서 다이어그램](docs/diagrams/sequence-login.png)

#### 3.3.2 공부 시작 및 종료 순서 다이어그램

사용자가 공부 시작 버튼을 누르면 프론트엔드는 공부 시작 API를 호출하고, 서버는 현재 진행 중인 세션 존재 여부를 확인한 뒤 새로운 `StudySession`을 생성한다. 사용자가 공부를 종료하면 서버는 종료 시각과 누적 공부 시간을 계산하여 세션 상태를 `completed`로 변경하고, 결과를 반환한다.

공부 시작 시 서버는 이미 진행 중인 세션이 있는지 확인하여 중복 세션 생성을 방지한다. 또한 공부 종료 시에는 세션 상태와 누적 시간을 반영하여 최종 공부 기록을 저장한다.

![공부 시작 및 종료 순서 다이어그램](docs/diagrams/sequence-study-session.png)

#### 3.3.3 목표 저장 및 조회 순서 다이어그램

사용자가 목표 시간을 입력하면 프론트엔드는 목표 저장 API를 호출한다. 서버는 해당 날짜의 목표 존재 여부를 확인한 후 새로운 목표를 저장하거나 기존 목표를 수정한다. 이후 목표 조회 또는 기록 조회 시에는 날짜별 최신 목표와 실제 공부 시간을 함께 계산하여 반환한다.

목표 저장 시 서버는 해당 날짜에 기존 목표가 존재하는지 확인한 뒤, 있으면 수정하고 없으면 새로 저장한다. 목표 기록 조회 시에는 날짜별 최신 목표와 실제 공부 시간을 비교하여 목표 달성 여부를 계산한다.

![목표 저장 및 조회 순서 다이어그램](docs/diagrams/sequence-daily-goal.png)

### 3.4 순서도

순서도는 특정 기능이 조건 분기와 상태 변화에 따라 어떻게 동작하는지 설명하기 위해 작성하였다.

#### 3.4.1 자동 일시정지 순서도

자동 일시정지 기능은 사용자가 공부 중일 때 브라우저 탭을 벗어나면 작동한다. 탭이 비활성화된 상태가 일정 시간 이상 유지될 경우 시스템은 현재 세션을 자동으로 일시정지하고 사용자에게 알림을 표시한다.

![자동 일시정지 순서도](docs/diagrams/flow-auto-pause.png)

#### 3.4.2 공부 세션 상태 전이도

공부 세션은 `active`, `paused`, `completed` 세 가지 상태를 기준으로 관리된다. 사용자는 시작 후 일시정지와 재개를 반복할 수 있으며, 최종적으로 종료 시 완료 상태로 전환된다.

![공부 세션 상태 전이도](docs/diagrams/flow-study-session.png)

상태 전이 흐름은 아래와 같다.

```text
active -> paused -> active -> completed
active -> completed
```

### 3.5 알고리즘 설계

본 프로젝트에서는 단순 CRUD 처리 외에도 실제 학습 흐름을 반영하기 위한 상태 관리와 데이터 보정 로직이 필요하였다. 따라서 알고리즘 설계에서는 자동 일시정지, 공부 시간 누적 계산, 목표 기록 조회, 중복 세션 방지 로직을 중심으로 정리하였다.

#### 3.5.1 자동 일시정지 알고리즘

자동 일시정지 기능은 브라우저의 탭 가시성 변화를 감지하여 실제 학습 시간만 기록하도록 설계하였다.

1. 사용자가 현재 공부 중인지 확인한다.
2. 브라우저 탭이 숨김 상태로 바뀌면 대기 타이머를 시작한다.
3. 일정 시간(30초) 이상 탭을 벗어난 상태가 유지되면 자동 일시정지를 수행한다.
4. 현재 세션의 상태를 paused로 변경하고 누적 시간을 저장한다.
5. 사용자에게 자동 일시정지 알림 메시지를 표시한다.


#### 3.5.2 공부 세션 시간 계산 알고리즘

공부 세션의 누적 시간은 시작 시각부터 종료 시각까지 단순 계산하지 않고, 일시정지와 재개 구간을 반영하여 계산하도록 설계하였다.


1. 공부 시작 시 세션 상태를 active로 저장한다.
2. 일시정지 시 마지막 재개 시각부터 현재 시각까지의 차이를 계산한다.
3. 계산된 시간을 기존 duration에 누적한다.
4. 재개 시 last_resume_time을 현재 시각으로 갱신한다.
5. 종료 시 active 상태였다면 마지막 활성 구간 시간을 추가 계산한다.
6. 최종 duration을 저장하고 상태를 completed로 변경한다.


#### 3.5.3 목표 기록 조회 알고리즘

목표 기록 조회 기능은 같은 날짜에 여러 번 목표가 수정될 수 있다는 점을 고려하여 날짜별 최신 목표를 기준으로 설계하였다.


1. DailyGoal 테이블에서 날짜별 최신 목표를 찾는다.
2. 같은 날짜의 StudySession 공부 시간 합계를 계산한다.
3. 목표 시간과 실제 공부 시간을 비교해 달성 여부를 계산한다.
4. 최근 기록 순으로 정렬하여 반환한다.


#### 3.5.4 진행 중 세션 중복 방지 알고리즘

공부 타이머 기능에서는 한 사용자가 동시에 여러 개의 진행 중 세션을 가지지 않도록 제약을 두었다. 이를 통해 실제 학습 흐름과 다른 중복 기록 생성을 방지하도록 설계하였다.


1. 사용자가 공부 시작 요청을 보낸다.
2. 서버는 해당 사용자의 StudySession 중 end_time이 없고 status가 active 또는 paused인 세션이 존재하는지 조회한다.
3. 진행 중 세션이 존재하면 새로운 세션 생성 요청을 거부한다.
4. 진행 중 세션이 없을 경우에만 새로운 StudySession을 생성한다.
5. 생성된 세션은 active 상태로 저장하고 이후 일시정지, 재개, 종료 흐름에 따라 관리한다.


## 4. 구현

### 4.1 구현 환경

본 프로젝트는 Windows 기반 개발 환경에서 진행하였고, 프론트엔드와 백엔드를 분리한 클라이언트-서버 구조로 구성하였다.

- 운영체제: Windows
- 프론트엔드 개발 환경: React 19, react-scripts 5
- 백엔드 개발 환경: Node.js, Express 5
- 데이터베이스: MariaDB
- 시스템 구조: 클라이언트(React) / 서버(Express) / 데이터베이스(MariaDB)

프론트엔드는 사용자 인터페이스와 화면 전환을 맡고, 백엔드는 REST API를 통해 인증, 과목 관리, 공부 세션 관리, 통계 조회, 목표 관리 기능을 처리한다. 데이터는 MariaDB에 저장되며, 서버는 데이터 조회와 저장, 상태 관리, 통계 집계를 맡는다.

### 4.2 개발 언어 및 라이브러리

본 프로젝트는 JavaScript를 기본 언어로 사용하였고, 프론트엔드와 백엔드에서 필요한 라이브러리를 나누어 적용하였다.

- `JavaScript`

- `react-router-dom`
  - 로그인, 메인화면, 타이머, 통계, 목표 관리 등 페이지 라우팅 처리에 사용
- `react-scripts`
  - React 프로젝트 실행 및 빌드 환경 구성에 사용
- `Node.js`
  - 백엔드 서버 실행 환경으로 사용

  - REST API 서버 구현 및 라우팅 처리에 사용
- `jsonwebtoken`
  - JWT 발급 및 인증 처리에 사용

  - 사용자 비밀번호 해시 처리와 검증에 사용
- `mysql2`
  - MariaDB와의 연결 및 SQL 쿼리 실행에 사용
- `cors`
  - 프론트엔드와 백엔드 간 교차 출처 통신 허용에 사용

### 4.6 주요 API

인증이 필요한 API를 호출할 때는 로그인 후 발급받은 JWT 토큰을 `Authorization` 헤더에 포함해야 한다.

#### 인증(Auth)

- `POST /register` : 사용자의 이메일과 비밀번호를 입력받아 새로운 계정을 생성한다.
- `POST /login` : 사용자 인증을 수행하고, 성공 시 JWT 토큰과 사용자 정보를 반환한다.

#### 과목 관리(Subject)

- `GET /subjects` : 로그인한 사용자의 과목 목록을 조회한다.
- `POST /subjects` : 새로운 과목명을 입력받아 사용자별 과목을 생성한다.
- `DELETE /subjects/:id` : 선택한 과목을 삭제한다.

#### 공부 세션(Study Session)

- `POST /sessions/start` : 선택한 과목과 메모 정보를 받아 새로운 공부 세션을 생성하고 타이머를 시작한다.
- `POST /sessions/pause` : 진행 중인 공부 세션을 일시정지하고 현재까지의 누적 공부 시간을 반영한다.
- `POST /sessions/resume` : 일시정지된 공부 세션을 다시 활성 상태로 전환한다.
- `POST /sessions/end` : 진행 중인 공부 세션을 종료하고 최종 공부 기록을 저장한다.
- `GET /sessions` : 사용자의 전체 공부 기록 목록을 조회한다.

#### 통계(Statistics)

- `GET /statistics/daily` : 오늘 날짜 기준의 총 공부 시간을 조회한다.
- `GET /statistics/weekly` : 최근 7일간의 공부 시간을 날짜별로 집계하여 조회한다.
- `GET /statistics/heatmap?year=YYYY&month=M` : 특정 연도와 월 기준의 공부 시간 히트맵 데이터를 조회한다.

#### 목표 공부시간(Daily Goal)

- `GET /daily-goal` : 오늘 날짜의 목표 공부 시간을 조회한다.
- `PUT /daily-goal` : 오늘 목표 공부 시간을 저장하거나 기존 목표를 수정한다.
- `GET /daily-goal/history` : 최근 목표 기록과 목표 달성 여부를 함께 조회한다.

### 6.1 결과

- 본 프로젝트를 통해 회원 인증, 과목 관리, 공부 타이머, 목표 설정, 통계 조회 기능을 통합한 웹 기반 학습 관리 시스템을 구현하였다.
- 사용자는 회원가입과 로그인 이후 공부할 과목을 생성하고, 과목별로 공부 세션을 시작해 학습 시간을 체계적으로 기록할 수 있다.
- 공부 세션은 `active`, `paused`, `completed` 상태를 기준으로 관리되며, 일시정지와 재개 기능을 포함해 실제 학습 흐름이 자연스럽게 반영되도록 설계하였다.
- 타이머 종료 시에는 공부 기록이 자동으로 저장되어 사용자가 별도의 저장 과정을 거치지 않아도 학습 이력을 누적할 수 있다.
- 메인화면에서는 오늘 공부 시간, 목표 시간, 현재 상태, 월별 공부 히트맵을 함께 보여주어 하루 학습 현황과 전체 학습 패턴을 한눈에 확인할 수 있다.
- 통계 화면에서는 오늘 공부 시간과 최근 7일 평균 공부 시간을 제공하고, 최근 7일 학습량을 차트 형태로 시각화해 학습 흐름을 비교할 수 있다.
- 일일 목표 화면에서는 오늘 목표 공부 시간을 설정하고 최근 목표 기록 및 달성 여부를 함께 확인하여 계획과 결과를 비교할 수 있다.
- 또한 탭 이탈 감지와 자동 일시정지 기능을 추가해 실제 학습에 집중한 시간만 기록되도록 보완함으로써, 단순 타이머를 넘어선 집중 학습 보조 기능까지 구현하였다.
